Best Practices and Tips

Best Practices

  1. Build a skeleton of the global project first - Before distributing the work to different teams, it is advisable to construct the skeleton of the global model. The skeleton should hold an OPD or OPM thing for each model which is developed each team.

  2. Keep a separated Copy of your project - As the Import feature in Opcat is not kept as a live link between the projects, every change in the imported files will not be reflected in the global model. Thus, it is advisable to hold and work on the separate files. The global model should not be worked on as a regular model file.

  3. Save before importing - The import operation is not undoable. It is advisable to create a backup of the model before importing. Backup is easily done by saving the model under a different name.

  4. Measure before importing - As imported models are copied exactly as they were in the original model, a good idea will be to take a good look at the model you wish to import before performing import. If needed, change it to suit the new model.

  5. Importing into inzoomed things - It is very common to import into an in-zoomed thing (process or object). In order to make sure that the thing can contain the imported model, it is advisable to enlarge the zoomed-in thing before importing.
