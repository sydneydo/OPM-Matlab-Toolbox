Import Tutorial

In this tutorial we will go through the process opening a global model and importing a model into it. We will use the ATM example as our imported model.



Import Process Flow

  1. Open Global Model
  2. Select where the imported model will be placed
  3. Open Import Dialog
  4. Choose Imported Model
  5. Save current model (if it's a new model)
  6. Import Done


Open Global Model

First, we need to create a global model, i.e., a base model in which an imported model will be copied. Open the Global OPM Model in Opcat by clicking on it, or create a new model with the following objects in it.

Our global model consists of a Process which is connected by an agent and a object.



Select Import Place

Within a global model, the imported model could be placed in two different locations:

  • Within an in-zoomed thing (process or object). In order to import the model there, the user should choose the in-zoomed thing before importing.
  • Outside of the in-zoomed thing.
The imported model will be placed in the exact same position in which is was laid in the original model.
For this example, we shell import the ATM example into the in-zoomed process, named "Main Feature". Notice that the process is selected.

Tip: In Order to avoid graphical problems, we enlarge the in-zoomed process to cover as much of the OPD as possible. To make sure the ATM fits in the selected process we shall open the ATM model and make sure that it is small enough to be placed within hte main process.

Open Import Dialog

There are two ways to open the import menu.

  • Choose the Import Icon from the toolbar 
  • Open the Import menu in the System Menu 

Choose Imported model

In the import dialog window we choose the OPM file to be imported by clicking on the Browse button. The File Chooser dialog is opened and we are able to locate and select the imported model.



Saving the Global Model

In order preserve model integrity, Opcat requires models to be saved before performing import. If the model is not new it will be saved Automatically before performing the import. If the model is a new model and was not saved before, the save model dialog box appears, and the user is prompted to save the model
