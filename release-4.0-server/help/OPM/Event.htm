5. Procedural Links 

    5.3 Event Links

Event links denote process executions due to internal and external events. There are time-out exception events, process termination events, state entrance events, state change events and external events. 

Time-out exception links connect timed processes or states with processes that should be activated in cases the processes/states violate their time constraints.  Time-out exception links are symbolized by cross headed arrows.

After 3 minutes that Timed Processing runs, Exception Handling should be activated.
After 3 minutes that Object Status is in its timed state, Exception Handling should be activated.

Invocation links are procedural links between invoking processes and invoked ones. The termination of the invoking processes activates the invoked processes. Invocation links are denoted by flash arrows.



Invoking Processing termination triggers Invoke Processing.

 

State entrance event links connect between states and the processes which are triggered when the corresponding objects enter the specific states. The process may change the object state. State entrance event links are denoted by instrument links with "E" literal inside. If the triggered process changes the object state, then it is denoted by a consumption link with "E" literal along its head.

Left figure: When Object Status enters its before state, State Entrance Processing is activated. It does not change Object Status.

Right figure: When Object Status enters its before state, State Entrance Processing is activated. It also changes Object Status from before to after.

State change event links connect between objects and the processes which are triggered when there is a change of value or state in the corresponding objects. State change event links are denoted by instrument links with "E" literal inside. If the connected object is environmental then the links are called external links.
