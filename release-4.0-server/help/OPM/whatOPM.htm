2. What is OPM?

The most important concepts of OPM are Objects and Processes. Processes affect, generate, or comsume objects. A State represents a situation at which an object can be and is changed by processes. OPM entities are connected by structural relations or procedural links.

There are five types of  structural relations in OPM:

  • Aggregation - Participation

  • Generalization - Specialization

  • Featuring - Characterization

  • Classification - Instantiation

  • General Structural Relations.

There are three different categoriez of procedural links:

  • Enabling links - including agent, instrument, and conditional links.

  • Transformation links - including consumption, result, and effect links.

  • Event links - including invocation, time-out, state entrance, and external event links.

OPM Scaling Mechanisms provide for a set of tools for controlling and managing system complexity. Complexity management entails a tradeoff that must be balanced between two conflicting requirements: completeness and clarity. On one hand, completeness requires that the system details be stipulated to the fullest extent possible. On the other hand, the need for clarity imposes an upper limit on the level of complexity of each individual diagram and does not allow for a diagram that is too cluttered or loaded. OPM scaling mechanisms include Zooming-in/Zooming-out and Unfolding/Folding mechanisms.
