5. Procedural Links 

    5.2 Transformation Links

A transformation link is a procedural link that connects a process with an object that is transformed by occurrence of that process. This transformation can be of consumption, as in consumption links, state change, as in effect links, or generation, as in reuslt links.

A consumption link is a transformation link that connects a process with an object that is consumed and eliminated as a result of the occurrence of that process. Consumption links are denoted by arrows from the object to the process.
 

Cash Accepting consumes Cash.

 

An effect link is a transformation link that connects a process with an object that was affected by an occurrence of that process. The effect is in the object state. Effect links are denoted by bi-directional arrows. 


 

Transaction Executing affects Account.

 

A result link is a transformation link that connects a process with an object that is constructed as a result of an occurrence of that process. Result links are denoted by arrows from the process to the object.

Cash Dispensing yields Cash.
