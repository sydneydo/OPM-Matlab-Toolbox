Explanation:

 

              Object - the object name. It can contain few words, but every word must begin with a capital

                     letter. The last word in the object name can’t end with “ing”. But if you want the

           object name ends with “ing”, write the word “object” after the object name.

             For example, “Building object is physical.”

 

               Process - the process name. It can contain few words, but every word must begin with a 

                     capital letter. The last word in the process name must end with “ing”. If you want 

                    the process name ends without “ing”, write the word “process” after the process 

                     name.

                           For example:  “P process is physical.”

 

               Cardinality - the cardinality of the object.

 

               state - the state of the object. Must start with low case letter.

 

               time - the time string. The time has to be in the following order:

                     year -> month -> day -> minute -> second -> millisecond.

 

               env_phys - environmental - physical declaration. Note, that you can add this declaration

                     after  every object or process name.

          For example: “A, which is physical, relates to B, which is environmental.

 

 TYPE - the type of the object (integer, char, …). Note, that all the types must be written in the low case, and only Boolean in

        upper   case. In addition, there was added the type string, which is actually a char array of length 50 chars.

 

 scope - “private”, “protected”, or “public”.
